# Деплой на Render.com с базой MySQL (AlwaysData или другая)

## В чём суть

Приложение на Render подключается к базе данных через **переменные окружения**. Сейчас в коде используются переменные `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `DB_PORT` (порт опционален). База может быть на AlwaysData или любом другом MySQL-хостинге.

## Шаг 1. Переменные окружения на Render

В [Render Dashboard](https://dashboard.render.com) → ваш сервис → **Environment** добавьте:

| Переменная   | Значение |
|-------------|----------|
| `DB_HOST`   | Хост MySQL с AlwaysData (например `mysql-[ваш-логин].alwaysdata.net`) |
| `DB_USER`   | Пользователь БД AlwaysData |
| `DB_PASSWORD` | Пароль пользователя БД |
| `DB_NAME`   | Имя базы данных |
| `DB_PORT`   | Порт MySQL на AlwaysData (часто **3306**, уточните в панели AlwaysData) |

Данные для подключения берутся в панели AlwaysData: **Bases de données** → ваша MySQL → параметры подключения.

Убедитесь, что в настройках сервиса на Render указан **Root Directory** (если API лежит в подпапке, например `api`).

## Шаг 2. Миграция базы (один раз)

Таблицы нужно создать в базе. Есть два варианта.

### Вариант A: Пустая база на AlwaysData

1. В панели AlwaysData создайте пустую базу MySQL и пользователя.
2. На Render в **Environment** задайте переменные (как в шаге 1).
3. Запустите миграции **один раз**:
   - либо через **Shell** на Render (если есть): зайти в контейнер и выполнить `npm run migrate`;
   - либо локально, задав те же переменные (например в `.env`) и выполнив в папке `api`:
     ```bash
     cd api && npm run migrate
     ```
4. По умолчанию миграция идёт с `force: false` — таблицы только создаются, данные не удаляются. Чтобы пересоздать таблицы (удалить данные), задайте `MIGRATE_FORCE=true` и снова запустите `npm run migrate`.

### Вариант B: Автоматический запуск миграций при деплое

В настройках сервиса Render в поле **Build Command** можно указать, например:

```bash
npm install && npm run migrate
```

Тогда при каждом деплое сначала выполнятся миграции (с `force: false`), затем старт приложения через **Start Command** `npm start`.

## Шаг 3. Старт приложения

**Start Command** на Render должен быть:

```bash
npm start
```

(или `node server.js`, если стартуете из корня репозитория — тогда путь к `server.js` должен быть правильным, например `node api/server.js`).

## Проверка

- В логах сервиса на Render должно появиться: «Подключение к базе данных успешно».
- Если видите ошибки подключения — проверьте `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `DB_PORT` и доступность MySQL с серверов Render (иногда хостинги разрешают подключения только с определённых IP; у AlwaysData это можно настроить).

## Кратко

1. Задать на Render переменные: `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, при необходимости `DB_PORT`.
2. Один раз выполнить миграции (`npm run migrate` в папке `api`), чтобы создать таблицы в базе.
3. Убедиться, что **Start Command** запускает приложение (`npm start` или `node server.js`).

После этого приложение на Render будет использовать вашу MySQL на AlwaysData (или другую), без переноса базы на Render.
